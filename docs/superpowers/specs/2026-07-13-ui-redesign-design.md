# UI Redesign Specification - 2026-07-13

本设计说明书旨在改进普通用户个人设置页面（`SettingsView.tsx`）与管理员后台管理页面（`AdminView.tsx`）的布局与交互体验，遵循干净、实用、信息密度的现代 SaaS 仪表盘设计规范。

## 1. 个人设置页面 (SettingsView.tsx) 优化

### 1.1 页面整体架构
* **布局**：采用 `grid grid-cols-1 md:grid-cols-3 gap-8` 的不对称双栏结构。
  * **左栏 (1/3 宽度)**：展示用户基本名片。包括圆形的用户名头像占位符、用户昵称、注册邮箱以及角色标识徽章（超级管理员、管理员、普通用户）。
  * **右栏 (2/3 宽度)**：依次垂直堆叠展示“基本资料修改”、“社交账号绑定”、“Passkey 安全密钥管理”三个功能区块。
* **圆角与边框**：所有区块均将圆角限制在 `rounded-lg` (8px)，去除多余的阴影和渐变悬浮感，改用极简的暗色平铺背景与细微边线（`bg-slate-900/40 border border-slate-800`）。

### 1.2 社交绑定列表与图标圆形化
* **图标样式**：第三方登录图标统一为 `rounded-full` 并且容器背景为透明，避免原图背景为透明时在白背景下显得突兀或显示为白色方块。对于没有图标的第三方应用，使用显示名首字母的圆形灰色占位符。
* **绑定与解绑按钮**：与账号名称在一行内左右两端对齐，并使用清晰的交互按钮样式。

### 1.3 Passkey 区块优化
* **添加按钮**：将“添加 Passkey”按钮移至“Passkey”区块头部的右侧，而不是堆叠在下方，使界面更紧凑。
* **密钥列表**：已绑定的密钥列表使用两端对齐，展示密钥自定义名称、设备类型、备份状态及创建时间，右侧为“删除”按钮。

## 2. 管理员后台管理页面 (AdminView.tsx) 优化

### 2.1 页面整体架构
* **布局**：引入**左侧固定侧边栏导航 + 右侧主内容区**。
  * **左侧侧边栏**（PC端固定宽度 `w-64`，移动端自动置顶堆叠）：包含系统 Logo/标识、主标题“管理员后台”、以垂直列表呈现的五个导航页签（全局设置、OAuth登录应用、邀请码、用户管理、DNS记录），底部为“返回主页”和“退出登录”链接。
  * **右侧主内容区**：根据当前选中的 Tab 渲染对应的内容。同样去除深色嵌套卡片，使用平铺无影的 8px 圆角卡片区块（`rounded-lg`）。
* **页签状态指示器**：当前选中的 Tab 采用高亮背景，且在左侧或底部带有绿色的指示边线，未选中项带有柔和的悬停（Hover）过渡动画。

### 2.2 各 Tab 内部布局优化
* **全局设置 (settings)**：
  * 将配置项划分为“注册通道模式”、“系统限制”和“邮件服务”三个子网格区块，所有表单输入框和 Toggle 复选框对齐。
* **OAuth 登录应用 (oauth)**：
  * “添加 OAuth 应用”的表单改用 2/3 列网格排版，模版选择移到最显眼的位置。
  * 列表中的“编辑应用”模块移出原有的表格行内 `<details>` 展开形式，在行下方渲染一个平铺的编辑表单，使输入框有足够的宽度，方便管理员修改配置。
* **邀请码管理 (invites)**：
  * 表格密度更高。状态标签（如“已使用”、“未使用”、“已失效”）使用不同颜色的圆角轻量级徽章，清晰易读。
* **用户管理 (users)**：
  * **synthetic 邮箱缩略显示**：对于使用保留地址/第三方平台生成的无邮箱临时长邮箱地址，在表格中自动截断展示，并在悬浮时显示完整邮箱地址，避免撑开表格导致排版错乱。
  * 角色管理与删除操作集中在操作列中，布局工整。
* **DNS 记录 (dns)**：
  * DNS 记录表格的“主机名”和“目标服务器”支持一键点击复制，删除操作带有红色的醒目警告色，并且交互对齐。

## 3. 技术实现与样式规范
* **Tailwind CSS**：严格使用 Layout 文件中已配置的系统字族和 `brand-600` 色值，圆角控制在 `rounded-lg` 以内。
* **交互细节**：为所有按钮、输入框、Tab 切换项添加平滑的 Hover 态、Focus 态与 Active 缩放过渡效果。
